## **Project Overview**

This project is a web application that helps students and self-learners build structured mathematical notes as a graph of interconnected “definitions.” Each definition has a title, optional aliases, and a LaTeX-formatted description. Definitions are shown as nodes in a graph, with edges representing conceptual links.

The app automatically discovers relationships between definitions by scanning descriptions for known terms and turning them into clickable links. Users can open movable, resizable windows to read definitions while keeping the graph in view, and can save/load their work via JSON import/export.

---

## **Core Features**

### **1\. Create, Modify, and Delete Definitions**

Users can create, edit, and delete definitions consisting of a title, aliases, and a LaTeX description. Changes are immediately reflected in the graph and in all autogenerated links.

This is implemented as a React form component using CodeMirror for text input and LaTeX.js for rendering. A custom React hook (`useGraphView`) manages the central definitions list and mutation methods.

---

### **2\. Automatic Linking Between Definitions**

The app automatically links related definitions by scanning descriptions for existing titles and aliases, then wrapping matches in a custom LaTeX macro (`\deflink{...}{...}`). Users can also insert this macro manually.

The linking system uses an Aho–Corasick multi-pattern matching algorithm over all titles and aliases, implemented as pure functions in a dedicated linking module. When definitions change, link detection is re-run so that graph edges and clickable links stay in sync.

---

### **3\. Graph View of Definitions**

Definitions and their links are visualized as an interactive graph: nodes represent definitions, edges represent links. Users can pan and zoom the graph, right-click nodes for actions, and double-click nodes to open detail windows.

The graph is built with Cytoscape.js in a React `GraphView` component. It derives its nodes and edges from the shared state managed by `useGraphView`, ensuring the graph always matches the current definitions and linking data.

---

### **4\. Definition Detail Windows**

Double-clicking a node opens a movable, resizable window showing the LaTeX-rendered definition description. Multiple windows can be open at once, enabling side-by-side comparison of related concepts while the graph remains visible.

These windows are implemented with standard React components and hooks (no external windowing library). A custom hook tracks open windows and their positions/sizes, and integrates with Cytoscape.js node events for opening/closing behavior.

---

### **5\. Import and Export of Graph Data**

Users can export all definitions to a JSON file and import them later to restore the same graph. This allows saving work locally, backing up data, or moving notes between machines.

Export/Import uses browser file APIs to download and read JSON files. The JSON structure mirrors the internal definitions list from `useGraphView`. On import, data is validated and loaded into state; links and graph layout are then recomputed automatically.

---

### **6\. Search Bar for Definitions**

A search bar allows users to quickly find a definition by title or alias. When a match is found, the corresponding node is highlighted and the graph centers on it; if no match exists, the search UI indicates failure.

The search feature is implemented in React and operates directly on the definitions list. Highlighting and camera centering are done through Cytoscape.js APIs, enabling smooth navigation in larger graphs.

---

## **Architecture Summary**

* The app is a React single-page application organized by feature under `src/features`.

* `features/editor` contains the definition editor, CodeMirror integration, and LaTeX macro setup.

* `features/graph` provides the `GraphView` Cytoscape.js integration and hooks for user interactions.

* `features/graph/hooks/useGraphView.ts` is a central hook that manages definitions and exposes methods for modifications, as well as derived graph data (nodes/edges).

* `features/linking` contains pure functions for building the Aho–Corasick matcher and performing description text replacement with the custom `\deflink` macro.

* `features/nodes` contains the definition window components and window state management logic.

* `App.tsx` wires these pieces together, instantiating `useGraphView` once and passing its state and handlers to the editor, graph, windows, and search components.
